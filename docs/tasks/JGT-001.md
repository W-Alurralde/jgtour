## Sprint

Sprint 02

## Estado

## Objetivo

Definir la estructura inicial del proyecto JGTravel para garantizar escalabilidad, mantenibilidad y desacoplamiento entre módulos.

## Contexto

Luego del Sprint 01 se identificó la necesidad de reorganizar la estructura del proyecto para soportar múltiples dominios del negocio turístico, internacionalización e integraciones con APIs externas.

## Actividades realizadas

- Revisión de la estructura existente.
- Creación de nuevas carpetas.
- Organización por responsabilidades.
- Preparación para arquitectura Feature Driven.
- Separación entre componentes, páginas, hooks, contextos y utilidades.

## Entregables

- Nueva estructura de carpetas.
- Organización del proyecto.
- Preparación para futuras integraciones.

## Resultado

Proyecto preparado para comenzar la arquitectura técnica.
