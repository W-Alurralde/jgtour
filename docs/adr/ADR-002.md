# ADR-002
## Arquitectura desacoplada mediante API Gateway

**Estado:** Aprobado

## Contexto

El frontend necesita consumir múltiples proveedores externos (Amadeus, OpenWeather, Google Places, Mercado Pago, etc.).

Exponer directamente las API Keys desde React representa un riesgo de seguridad y dificulta el reemplazo futuro de proveedores.

## Decisión

React nunca consumirá APIs externas directamente.

Toda comunicación pasará por una API propia de JGTravel.


## Arquitectura

React

↓

API Gateway JGTravel

↓

## Servicios externos

Justificación
Protección de credenciales.
Desacoplamiento.
Mayor mantenibilidad.
Posibilidad de cambiar proveedores sin modificar el frontend.
Consecuencias

El frontend solamente conocerá la API propia de JGTravel.
