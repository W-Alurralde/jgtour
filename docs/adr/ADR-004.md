# ADR-004
## Estrategia de Sincronización de Datos

**Estado:** Aprobado

## Contexto

Las APIs gratuitas presentan limitaciones de consultas.

Durante eventos, ferias o demostraciones podrían existir cientos o miles de usuarios simultáneos.

Consultar proveedores externos por cada búsqueda resulta inviable.

## Decisión

Los datos se dividirán en dos categorías.

Datos estáticos

- destinos
- hoteles
- restaurantes
- imágenes
- paquetes

Estos datos serán sincronizados periódicamente mediante procesos automáticos.

Datos dinámicos

- disponibilidad
- precios finales
- reservas

Serán consultados únicamente cuando sea necesario.

## Justificación

Reduce costos.

Disminuye el consumo de APIs.

Mejora la velocidad de respuesta.

Permite escalar el sistema.

## Consecuencias

La plataforma dispondrá de una base de datos propia actualizada periódicamente.
