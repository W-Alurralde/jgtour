# ADR-005
## Dolibarr como Backoffice Administrativo

**Estado:** Aprobado

## Contexto

Desarrollar desde cero un CRM, ERP, sistema de pedidos, facturación y gestión administrativa incrementaría significativamente el tiempo y costo del proyecto.

Dolibarr ofrece estas funcionalidades mediante una plataforma Open Source consolidada.

## Decisión

Dolibarr será utilizado exclusivamente como Backoffice Administrativo.

React no accederá directamente a Dolibarr.

La comunicación se realizará mediante la API propia de JGTravel.

## Arquitectura

React

↓

API JGTravel

↓

Dolibarr

## Responsabilidades de Dolibarr
Clientes
CRM
Pedidos
Presupuestos
Facturación
Proveedores
Productos
Administración
## Justificación
Reduce meses de desarrollo.
Aprovecha una solución madura.
Permite reemplazar el ERP en el futuro sin afectar el frontend.
Centraliza la lógica administrativa.

## Consecuencias

El frontend permanece desacoplado del ERP.
